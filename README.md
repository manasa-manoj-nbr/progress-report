# Batch Progress Report Generator — TestCrack

Generates weekly batch progress reports for IELTS coaching institutes. An institute owner submits a batch ID and receives a full report with computed statistics, at-risk student classifications, per-student breakdowns, and an AI-generated narrative summary.

> **Note:** This feature uses the **Google Gemini API** (Gemini) for AI narrative and WhatsApp message generation. The project is configured to use `GEMINI_API_KEY`.

## Project Structure

```
progress-reporter/
├── backend/
│   ├── server.js                  # Express API server
│   ├── services/
│   │   ├── reportService.js       # Data loading, all arithmetic, at-risk classification
│   │   └── aiService.js           # AI calls for narrative + WhatsApp messages (Gemini only)
│   ├── data/
│   │   └── mock_students.json     # Mock student data loaded from disk
│   ├── .env                       # API key (not committed)
│   └── .env.example               # Template for environment variables
└── frontend/
    └── src/                       # Vite + React + TailwindCSS dashboard UI
```

## Setup

### 1. Configure API Key

```bash
cp backend/.env.example backend/.env
```

Add your Claude API key in `backend/.env`:

```
ANTHROPIC_API_KEY=your_actual_key_here
GEMINI_API_KEY=your_actual_key_here
```

### 2. Install Dependencies

```bash
cd backend && npm install
cd ../frontend && npm install
```

### 3. Run

```bash
# Terminal 1 — Backend (port 3001)
cd backend
node server.js

# Terminal 2 — Frontend (port 5173)
cd frontend
npm run dev
```

## Design Decisions

| Decision                                  | Rationale                                                                                      |
| ----------------------------------------- | ---------------------------------------------------------------------------------------------- |
| All arithmetic in `reportService.js`      | No AI involvement in numeric calculations                                                      |
| AI receives pre-computed stats only       | Cost efficiency and prompt engineering best practice                                           |
| Three independent at-risk rules           | Rule 1: inactive ≥5 days · Rule 2: any band declined · Rule 3: exam ≤30d AND >1.5 below target |
| `tutor_alert_message` per at-risk student | WhatsApp-ready message generated by AI, scoped to each student's context                       |
| Weekly report date derivation             | Defaults to the day after the latest `last_active` date in the batch                           |
| Student data loaded from disk             | `mock_students.json` — no database required                                                    |
| Claude-first AI integration               | Falls back to Gemini only when Claude credentials are unavailable                              |
